# Tarka Quiz — Implementation Guide
> Nepal ko sabai bhanda tez trivia · Real-time multiplayer quiz

---

## What is this project?

Tarka Quiz is a real-time multiplayer trivia game about Nepal. Players join a room using a code, answer 10 questions simultaneously, and compete on a live leaderboard. Questions are generated by Claude AI and cached in Redis. Built with Next.js (frontend) + FastAPI (backend) + WebSockets.

## How the game works (data flow)

```
1. Player opens localhost:3000
   └── Sees JoinScreen — fills name, tole, category

2. Clicks "Create room"
   └── Frontend generates room code e.g. TARK42
   └── Opens WebSocket: ws://localhost:8000/ws/TARK42/player-abc
   └── Sends: { name: "Aarav", tole: "Patan" }

3. Server receives connection
   └── Creates Room(code="TARK42", host="player-abc")
   └── Sends back: { event: "joined", is_host: true, players: [...] }
   └── Frontend shows LobbyScreen

4. Friend opens localhost:3000, enters code TARK42
   └── Opens WebSocket: ws://localhost:8000/ws/TARK42/player-xyz
   └── Server adds to room, broadcasts player_joined to everyone
   └── Both lobbies update live

5. Host clicks "Start game"
   └── Frontend sends: { event: "start_game", category: "Nepal general" }
   └── Server fetches 10 questions from Redis cache
   └── Broadcasts: { event: "countdown", tick: 3 }
   └── Broadcasts: { event: "countdown", tick: 2 }
   └── Broadcasts: { event: "countdown", tick: 1 }
   └── All clients show CountdownScreen

6. Server broadcasts question
   └── { event: "question", text: "...", options: [...], time_limit: 20 }
   └── NO correct answer included — stays secret until timer ends
   └── All clients show QuestionScreen with 20s timer

7. Player picks an answer
   └── Frontend sends: { event: "answer", answer: "C" }
   └── Server checks answer, calculates speed bonus points
   └── Sends ONLY to that player: { event: "answer_ack", correct: true, points: 850 }
   └── Other players don't know this player's answer yet

8. Timer ends (or all players answered)
   └── Server reveals: { event: "round_result", correct_answer: "C",
                         explanation: "...", leaderboard: [...] }
   └── All clients show ScoringScreen for 3 seconds

9. Repeat steps 6-8 for all 10 questions

10. After Q10
    └── Server sends: { event: "game_over", final_leaderboard: [...] }
    └── All clients show GameOverScreen
```

## WebSocket events reference

### Client → Server

| Event | Fields | When |
|-------|--------|------|
| *(join payload)* | `name`, `tole` | First message after WS opens |
| `start_game` | `category` | Host clicks Start in lobby |
| `answer` | `answer` ("A"/"B"/"C"/"D") | Player picks during question |

### Server → Client

| Event | Fields | Sent to |
|-------|--------|---------|
| `joined` | `room_code`, `is_host`, `players[]` | Joining player only |
| `player_joined` | `player {id, name, tole}` | Everyone in room |
| `player_left` | `player_id` | Everyone in room |
| `countdown` | `tick` (3, 2, 1) | Everyone in room |
| `question` | `index`, `total`, `text`, `options[]`, `category`, `time_limit` | Everyone in room |
| `answer_ack` | `correct`, `points`, `your_total` | Answering player only |
| `round_result` | `correct_answer`, `explanation`, `leaderboard[]` | Everyone in room |
| `game_over` | `final_leaderboard[]` | Everyone in room |

## Score calculation

```python
# Max 1000 points per question
# Speed bonus: answer instantly = 1000pts, answer at last second = 500pts
# Wrong answer = 0pts

speed_factor = max(0, 1 - (answer_time_seconds / time_limit))
points = int(500 + 500 * speed_factor)  # range: 500-1000
```

## Backend files to build next

### `server/models.py`
Dataclasses for Player, Question, Room state enum.
No logic — just data shapes.

### `server/questions.py`
- Hardcoded 20 Nepal questions (to start, no Redis needed)
- `get_questions(category, n)` → returns list of Question
- Later: Redis cache + Claude API generation

### `server/room.py`
The state machine. States: lobby → countdown → question → scoring → gameover
- `Room.join(player)` → adds player, broadcasts player_joined
- `Room.start_game()` → runs countdown, starts question loop
- `Room.receive_answer(player_id, answer)` → scores, sends answer_ack
- `Room.broadcast(event)` → sends JSON to all connected players

### `server/main.py`
- FastAPI app with CORS
- `@app.websocket("/ws/{room_code}/{player_id}")` → the single WS endpoint
- Room registry: `dict[str, Room]` in memory
- Reads first message as join payload, then loops reading events

Once backend is running, test this sequence:

- [ ] Open two browser tabs on localhost:3000
- [ ] Tab 1: Create room — should see lobby with room code
- [ ] Tab 2: Enter same room code — both tabs should show 2 players
- [ ] Tab 1 (host): Click Start — both tabs countdown 3-2-1
- [ ] Both tabs: See the same question simultaneously
- [ ] Answer in tab 1 — see correct/wrong immediately
- [ ] Timer ends — both tabs see scoring screen
- [ ] After 10 questions — both tabs see game over
